Week 3: SQL & Data Querying

Overview
In real-world environments, data is stored in relational databases. Your ability to extract,
manipulate, and analyze data using SQL is critical for solving business problems efficiently.

Objective

To develop strong SQL querying skills by:
• Extracting and transforming structured data
• Writing efficient queries
• Solving business-driven analytical questions
Datasets (Please kindly note that you are required to use both
datasets, making it a total of 2 tasks for this week)

Chinook Database (Music Store Data)
https://github.com/lerocha/chinook-database

Sales Dataset (AdventureWorks Lite / Equivalent)
https://www.kaggle.com/datasets/kyanyoga/sample-sales-data

Task Breakdown

• 1. Database Setup
Import datasets into a SQL environment (PostgreSQL/MySQL)
Understand schema relationships (tables, keys, constraints)

• 2. Core SQL Queries
Write queries using:
SELECT, WHERE, ORDER BY
GROUP BY, HAVING
Aggregate functions (SUM, AVG, COUNT)

• 3. Advanced SQL Concepts
• Joins (INNER, LEFT, RIGHT)
Subqueries
• Window functions (ROW_NUMBER, RANK, PARTITION BY)

• 4. Business Problem Solving
Answer key analytical questions such as:
• Top-performing products or customers
Revenue trends over time
• Customer purchasing behavior

• 5. Query Optimization
Improve performance using indexing concepts
Write clean and readable SQL
Tools Requirement
PostgreSQL / MySQL
SQL client (DBeaver, pgAdmin, etc.)
Deliverables
• SQL Script File (.sql)
• Document explaining queries and insights
